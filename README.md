# 基于多智能体协同的混合量化交易回测系统

本项目是一个融合了**传统技术指标分析**、**大语言模型（LLM）基本面情感分析**、**计算机视觉（CNN）K线形态识别**以及**多智能体（Multi-Agent）协同决策**的混合量化交易回测系统。

系统通过多个专业化智能体（Agent）分别评估市场的技术面、基本面和风险度，最终由“首席决策官”（Meta-Agent）整合所有维度的报告，做出交易决策（`BUY` / `SELL` / `WAIT`），并在历史数据中进行双向（多头与空头）回测验证。

---

## ─── 系统架构 ───

系统采用分层多智能体协同架构。在每一个回测时间步，系统流程如下：

```text
                    ┌────────────────────────┐
                    │    历史行情与新闻数据  │
                    └───────────┬────────────┘
                                │
         ┌──────────────────────┼──────────────────────┐
         ▼                      ▼                      ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│ 趋势温度智能体   │    │ 基本面新闻智能体│    │   风控智能体    │
│ (或CNN视觉智能体)│    │ (本地/API RAG)  │    │  (最大回撤监控)  │
└────────┬────────┘    └────────┬────────┘    └────────┬────────┘
         │                      │                      │
         │ (技术面报告)         │ (基本面情感报告)     │ (风控指令报告)
         └──────────────┬───────┴──────────────────────┘
                        ▼
            ┌───────────────────────┐
            │     首席决策官        │  <-- DeepSeek-V3 / Chat
            │  (Meta-Decision Agent)│
            └───────────┬───────────┘
                        │ (JSON: BUY / SELL / WAIT)
                        ▼
            ┌───────────────────────┐
            │       回测引擎        │  <-- 执行交易、双向持仓、
            │   (Backtest Engine)   │      动态移动止盈止损
            └───────────────────────┘
```

## ─── 环境准备与安装 ───

### 1. 克隆项目并创建虚拟环境
推荐使用 Python 3.10+ 版本。
```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境 (Windows)
.venv\Scripts\activate

# 激活虚拟环境 (macOS/Linux)
source .venv/bin/activate
```

### 2. 安装依赖项
```bash
pip install -r requirements.txt
```

### 3. 配置环境变量
在项目根目录下创建一个名为 `.env` 的文件，填入您的 API 密钥：
```ini
DEEPSEEK_API_KEY="您的_DEEPSEEK_API_KEY"
# FINNHUB_API_KEY="您的_FINNHUB_API_KEY" (可选，用于降级新闻检索)
```
> **注意：** 项目中已通过 `.gitignore` 确保 `.env` 文件不会被提交到代码仓库中，以保护密钥安全。

## ─── 数据准备 ───

系统优先从本地数据源读取历史行情和新闻，以避免回测过程中的高频 API 请求限制和网络延迟。

### 1. 下载行情数据
运行数据下载脚本，通过 `akshare` 接口下载英伟达（NVDA）等美股的真实日线数据。
```bash
python download_data.py
```
这将在 `data/` 目录下生成 `105.NVDA_daily.csv`。

### 2. 下载历史新闻数据
为了支持基本面分析，运行新闻检索脚本，利用百度资讯检索目标时间段内的相关新闻，并执行**全局标题去重**和**前瞻偏差过滤**。
```bash
python download_news.py
```
这将在 `data/` 目录下生成 `NVDA_news.csv`。

## ─── 运行回测 ───

执行主程序即可启动多智能体协同回测流程：
```bash
python main.py
```

### 运行机制说明：
1. **滑动窗口初始化**：主程序将自动加载数据，并从第 30 个时间步开始进行滑动窗口评估（预留前 30 天数据以供技术指标计算）。
2. **多智能体并行分析**：
   - **趋势/视觉智能体**：对过去 30 天的市场趋势做出买卖倾斜评估。
   - **新闻分析智能体**：对回测当前日期前 30 天的新闻进行 RAG 检索，提取重要信息并使用 DeepSeek 给出【利好/利空/中性】的情绪评估。
   - **风控智能体**：监控价格相较于 10 日高点的回撤状态。
3. **首席决策官裁决**：将上述三份报告汇总至 `Meta-Decision Agent`，在降低 Temperature 且开启 JSON Mode 的状态下输出严格格式化的交易指令。
4. **交易执行与资金更新**：回测引擎根据指令买入、卖出或观望，并输出实时日志。

## ─── 核心智能体设计 ───

### 1. 技术面智能体（双策略可选）
*   **趋势温度分析师 (`TrendTemperatureAgent`)**：
    基于经典动量思想。综合“均线多头排列度（EMA 5/10/20）”与“唐奇安通道位置（20日最高/最低点）”，计算 0-100 的温度评分。温度高于 70 判定为 `BUY`，低于 30 判定为 `SELL`，其余判定为 `WAIT`。
*   **视觉分析师 (`CNNVisionAgent`)** *(可选)*：
    利用 `mplfinance` 将行情窗口实时绘制成 K 线图，保存至内存缓冲区（避免频繁落盘 I/O 导致性能下降），然后输入预训练的 CNN 模型 (`cnn_model.h5`) 进行分类（0=看跌，1=看涨，2=观望）。

### 2. 基本面新闻分析师 (`FundamentalNewsAgent`)
通过本地的 RAG 机制获取特定标在当前交易日前 30 天内的真实历史新闻。大模型基于新闻上下文评估情绪，并严格遵守逻辑，若无有效新闻则默认输出中性报告，防止大模型由于缺乏信息而产生幻觉或盲目给方向。

### 3. 风控管理员 (`RiskControlAgent`)
负责防范异常破位风险。如果当前价格距离 10 日内最高价的回撤比例超过了硬性阈值（默认 15%），该智能体将输出一票否决式的“空仓避险”强制指令。

### 4. 首席决策官 (`MetaDecisionAgent`)
基于 DeepSeek API 驱动，担任整个决策链路的核心。其逻辑规则具有严格的优先级限制：
- **风控至上**：若风控智能体提示破位，不论其他智能体信号如何，一律输出 `WAIT`（空仓避险）。
- **顺势而为**：技术面与基本面方向达成共识（例如技术强烈看涨且基本面无利空）时，才允许输出 `BUY` 或 `SELL`。
- **意见分歧则观望**：技术与基本面冲突，或者技术面呈震荡态势时，输出 `WAIT`。

## ─── 回测引擎特性 ───

回测引擎具有以下专为实际交易场景设计的机制，力求模拟真实的交易环境：

*   **双向交易支持**：不仅支持建立多头头寸（`BUY`），同样支持建立空头头寸（`SELL`）进行做空套利。
*   **双向移动追踪止盈 (Trailing Stop)**：
    - 多头持仓下：若价格从开仓后的历史最高点回撤超过 12%，强制平仓止盈。
    - 空头持仓下：若价格从开仓后的历史最低点反弹超过 12%，强制平仓止盈。
*   **硬性止损 (Hard Stop Loss)**：
    - 针对价格突变，设定了 10% 的硬性止损线（多头跌破 10%，空头冲高 10%），以锁定最大单笔亏损。
*   **交易费用模拟**：引入单边交易费率（默认 0.1%），确保回测收益扣除摩擦成本，更接近实盘表现。
